# Security Verification Results

Use this section to report security-related verification evidence when the approved workflow includes dependency checks, secret scanning, code scanning, or other security review activities.

## Required Coverage

Summarize the following when security verification was part of the project workflow:

- the security checks or scanning tools used by the project
- the sprint, release, repository area, or deployment stage covered by the reported checks
- the important findings, including whether they were resolved, accepted, or remained open
- the practical meaning of the findings for repository hygiene, deployment readiness, or technical limitations
- what the reported security checks did not cover

If formal security verification was not part of the approved workflow, state that directly and identify the approved alternative assurance basis.

## Interpretation Guide

Use a short narrative after the table to explain:

- what the reported security evidence actually verified
- what important risks were found or ruled out
- what unresolved issues, accepted risks, or workflow limitations remained at the time of reporting

## Traceability Reminder

Keep this section consistent with repository alerts, workflow history, sprint appendices, and the validation plan. Do not imply a full security assessment unless the project actually performed one.

## Writing Reminder

Keep the discussion proportional to the evidence gathered. This section should show practical verification results, not generic security theory.
