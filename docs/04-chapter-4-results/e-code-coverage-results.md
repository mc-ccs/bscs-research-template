# Code Coverage Results

Use this section when the approved workflow includes automated tests with coverage reporting.

## Required Coverage

Summarize the following when code coverage tooling is part of the project workflow:

- the coverage tool or report source used by the project
- the sprint, module, release, or branch covered by the reported metric
- the reported coverage value or range
- the scope of the covered tests
- important uncovered areas that affect interpretation

If code coverage was not part of the approved workflow, state that directly and identify the approved replacement evidence used instead.

## Interpretation Guide

After the table, explain:

- how the coverage evidence supports automated verification
- whether the reported coverage is adequate for the approved workflow
- what important limitations remain because of uncovered code paths or excluded modules

## Traceability Reminder

Keep this section aligned with the automated testing workflow, CI/CD evidence, and sprint appendices. If the coverage basis changed across sprints, make that change explicit.

## Writing Reminder

Coverage is supporting evidence, not the full validation basis by itself. Keep the claims limited to what the coverage report actually shows.
