# Objectives and RQ Alignment

Use this section to state the project objectives in a way that aligns directly with the institutional research questions.

## Required Alignment

The standard objective pattern is:

- **RQ1 - Requirements Engineering**: analyze or define the requirements needed to address the problem
- **RQ2 - System Implementation**: design and implement the MVP or core artifact
- **RQ3 - System Validation**: validate the developed artifact using measurable evidence

## Suggested Format

| Objective | Related RQ | Notes on Measurement or Output |
| --- | --- | --- |
| To analyze and define the functional and non-functional requirements of the proposed system. | RQ1 | Requirements basis, user stories, backlog, or design inputs |
| To design and implement an MVP that addresses the identified problem. | RQ2 | Architecture, core logic, implemented features |
| To validate the developed artifact using testing, user evaluation, or other approved technical metrics. | RQ3 | Functional testing, SUS, SEQ, accuracy, performance, or similar evidence |

## Writing Reminder

Use action verbs such as analyze, design, implement, and validate. Avoid vague objectives such as "to know," "to understand," or "to determine" unless they lead to a concrete engineering output.
