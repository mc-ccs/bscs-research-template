# Project Context

Use this section to explain the real-world setting of the project and the concrete problem that motivated the study.

## Required Coverage

Describe the following in a concise engineering style:

- the organization, community, or user environment where the problem occurs
- the target users or stakeholders affected by the problem
- the current workflow, current system, or current practice
- the specific pain points, inefficiencies, risks, or limitations in that current condition
- why a software-based solution is appropriate for the problem

## Standard Paragraph Flow

### 1. Operational Context

Briefly explain the domain and where the problem occurs.

### 2. Stakeholders and Current Practice

Identify the primary users and summarize how the work is currently done.

### 3. Problem Statement Basis

State the concrete pain points that justify the project. Use observable issues such as delays, errors, repeated manual work, poor traceability, low accessibility, or weak decision support.

## Writing Reminder

Avoid broad history, generic industry background, or solution details that belong in later sections. The goal of this section is to make the problem understandable and credible.
